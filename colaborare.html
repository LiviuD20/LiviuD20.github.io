Flux de Lucru (Workflow) în Echipă

Unul dintre cele mai populare și sigure fluxuri de lucru se numește "GitHub Flow" sau "Feature Branch Workflow". Acesta previne modificările directe pe ramura `main`.

[Image of a feature branch workflow diagram]

Pașii de Urmat:

  1. 1. Sincronizează-te cu `main`
    Asigură-te că ai cea mai nouă versiune a proiectului. Mergi pe ramura `main` și rulează `git pull` pentru a descărca orice modificări făcute de colegi.
    Sincronizare
  2. 2. Creează o ramură nouă (Branch)
    Nu lucra niciodată direct pe `main`. Creează o ramură nouă, descriptivă, pentru sarcina ta. (Ex: `git checkout -b feature/formular-contact`).
    Ramificație
  3. 3. Lucrează și fă 'Commits'
    Adaugă cod, modifică fișiere și salvează-ți progresul des cu `git commit -m "mesaj descriptiv"`. Munca ta este izolată pe ramura ta.
    Dezvoltare
  4. 4. Deschide un 'Pull Request' (PR)
    Când ai terminat, încarcă ramura pe GitHub (`git push origin feature/formular-contact`) și deschide un Pull Request. Acesta este o cerere de a-ți îmbina munca în ramura `main`.
    Revizuire
  5. 5. Revizuirea Codului (Code Review)
    Colegii tăi se vor uita peste codul din Pull Request. Ei pot lăsa comentarii, pot cere modificări sau pot aproba PR-ul. Acesta este cel mai important pas pentru a menține calitatea codului.
    Revizuire
  6. 6. Îmbină (Merge) și Șterge
    După ce PR-ul este aprobat, modificările sunt îmbinate (merged) în `main`. Ramura ta veche (`feature/formular-contact`) poate fi ștearsă. Ciclul se reia de la pasul 1.
    Integrare

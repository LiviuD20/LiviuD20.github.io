Concepte Fundamentale Git

Pentru a lucra eficient, trebuie să înțelegi "vocabularul" de bază al Git.

Un Repository (sau "repo") este folderul proiectului tău. Git stochează tot istoricul modificărilor în acest folder (într-un subfolder ascuns numit `.git`).
  • Local Repository: Este copia de pe computerul tău, unde lucrezi și faci 'commits'.
  • Remote Repository: Este copia de pe un server (ca GitHub), unde îți sincronizezi munca și colaborezi cu echipa.

Un Commit este o "fotografie" (snapshot) a proiectului tău la un anumit moment. Este unitatea de bază a istoricului în Git. Când ești mulțumit de modificările făcute (ex: ai adăugat o funcție, ai reparat un bug), faci un 'commit' cu un mesaj descriptiv (ex: "Am adăugat formularul de contact").

O Ramură (branch) este o linie independentă de dezvoltare. Ramura principală se numește de obicei `main`. Când vrei să lucrezi la ceva nou (ex: o nouă pagină "Despre Noi"), creezi o ramură nouă (ex: `feature/pagina-despre`). Poți lucra pe această ramură fără a afecta codul stabil de pe `main`.

Merge este procesul prin care aduci modificările de pe o ramură (ex: `feature/pagina-despre`) înapoi în altă ramură (ex: `main`). Acesta este momentul în care funcționalitatea ta nouă devine parte din proiectul principal.

Aceste comenzi sincronizează repo-ul tău local cu cel remote (de pe GitHub):
  • `git push` (Împinge): Trimite 'commits'-urile tale locale către serverul remote.
  • `git pull` (Trage): Descarcă modificările făcute de alții de pe serverul remote și le îmbină cu copia ta locală.

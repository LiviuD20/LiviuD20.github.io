Studiu de Caz: Importanța utilizării GIT / GitHub în lucru de echipă

Cum organizezi colaborarea pe un proiect comun, folosind bune practici, comenzi Git și hostare pe GitHub Pages.

Introducere

GIT este un sistem de control al versiunilor distribuit, iar GitHub este o platformă web care găzduiește repository-uri Git și oferă facilități pentru colaborare (pull requests, issues, actions). Folosirea acestora în echipă reduce conflictele, îmbunătățește trasabilitatea modificărilor și crește productivitatea.

Beneficii principale

  • Istoric clar al codului — fiecare modificare are autor și mesaj
  • Ramuri (branches) pentru funcționalități noi sau bugfix-uri
  • Code review prin Pull Requests
  • Integrare continuă / Deploy automat cu GitHub Actions
  • Rezolvarea conflictelor și revenire la versiuni anterioare

Flux recomandat de lucru (exemplu)

  1. Clonare repo: git clone
  2. Creare branch pentru o sarcină: git checkout -b feat/noua-functie
  3. Commit frecvent cu mesaje clare: git add . && git commit -m "descriere scurtă"
  4. Push pe remote: git push origin feat/noua-functie
  5. Deschidere Pull Request pe GitHub, review, merge după aprobare

Exemplu de reguli bune (convenții)

  • Branch-uri: main (sau master) - stabil, develop - integrare, feat/*, fix/*
  • Mesaje commit: prefix scurt (feat:, fix:, docs:), urmat de descriere
  • Pull Request-uri mici, cu descriere și link la issue
  • Folosirea .gitignore pentru fișiere locale (node_modules, .env, etc.)

Titlu: Pași practici pentru cerințele tale

Mai jos sunt comenzile minimal necesare pentru a crea repo local, a încărca pe GitHub și a hosta pagina cu GitHub Pages.

Comenzi Git (Git Bash)

# 1. Initializează repo local
git init

# 2. Adaugă fișiere și commit inițial
git add index.html
git commit -m "Initial commit: adaug index.html pentru studiul de caz"

# 3. Creează branch principal (opțional)
git branch -M main

# 4. Adaugă remote (înlocuiește  și repo-url)
git remote add origin https://github.com/<username>/studiu-de-caz.git

# 5. Push pe GitHub
git push -u origin main

Hostare pe GitHub Pages

  • Creează repository public pe GitHub cu numele studiu-de-caz.
  • După push, mergi la Settings → Pages → Branch: selectează main și root (/) → Save.
  • Link-ul final va fi: https://<username>.github.io/studiu-de-caz/ (înlocuiește <username> cu username-ul tău GitHub).

Rezumat rapid

  • Nume repo: studiu-de-caz
  • Fișier principal: index.html
  • Hostare: GitHub Pages
  • Link rezultat: https://<username>.github.io/studiu-de-caz/

Sfat util

Fă commit-uri mici și descriptive. Folosește Pull Requests pentru review — asta protejează codul și învață echipa.
